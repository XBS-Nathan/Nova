# Interactive Selectors for `nova init`

## Summary

Replace the free-text prompts in `nova init` with interactive select lists powered by `pterm.DefaultInteractiveSelect`. Version options (PHP, Node, MySQL, Postgres, Redis) are fetched from Docker Hub at build time via `go generate` and embedded as Go slices.

## Version Fetching (`go generate`)

### Generator program

New file: `internal/config/cmd/fetchversions/main.go`

A Go program invoked by `//go:generate` that:

1. Queries Docker Hub API (`https://hub.docker.com/v2/repositories/library/{image}/tags`) for each image
2. Filters tags to meaningful versions:
   - **PHP**: `X.Y-fpm` tags -> extract `X.Y`, dedupe, sort descending (e.g. `8.4, 8.3, 8.2, 8.1`)
   - **Node**: `X-alpine` tags -> extract major `X`, dedupe, sort descending (e.g. `22, 20, 18`)
   - **MySQL**: `X.Y` semver tags (no suffix), dedupe, sort descending (e.g. `9.0, 8.0`)
   - **Postgres**: bare integer tags (major versions), sort descending (e.g. `17, 16, 15`)
   - **Redis**: bare integer or `X.Y` tags, sort descending (e.g. `8, 7`)
3. Writes `internal/config/versions_generated.go`

### Fallback

If Docker Hub is unreachable, the generator falls back to hardcoded defaults matching the existing `Default*` constants so the build never fails.

### Generated file

`internal/config/versions_generated.go` (committed to git):

```go
// Code generated by fetchversions; DO NOT EDIT.
package config

var PHPVersions = []string{"8.4", "8.3", "8.2", "8.1"}
var NodeVersions = []string{"22", "20", "18"}
var MySQLVersions = []string{"9.0", "8.0"}
var PostgresVersions = []string{"17", "16", "15"}
var RedisVersions = []string{"8", "7"}
```

### Generate directive

New file: `internal/config/gen_versions.go`

```go
package config

//go:generate go run ./cmd/fetchversions
```

### Build workflow

```bash
go generate ./internal/config/...  # refresh versions from Docker Hub
go build ./...                      # build with embedded versions
```

## Interactive `nova init`

### Selector fields

| Field | Options source | Default selection |
|---|---|---|
| Project type | `["laravel", "other"]` (hardcoded) | auto-detected or `other` |
| PHP version | `config.PHPVersions` (generated) | `config.DefaultPHP` |
| Node version | `config.NodeVersions` (generated) | `config.DefaultNode` |
| Package manager | `["npm", "yarn", "pnpm"]` (hardcoded) | `config.DefaultPackageManager` |
| DB driver | `["mysql", "postgres"]` (hardcoded) | `"mysql"` |
| DB version | `config.MySQLVersions` or `config.PostgresVersions` (dynamic, based on selected driver) | `config.DefaultMySQLVersion` / `config.DefaultPostgresVersion` |
| Redis version | `config.RedisVersions` (generated) | `config.DefaultRedisVersion` |

### Free-text fields

| Field | Default |
|---|---|
| Domain | `{projectName}.test` |
| DB name | sanitized project name |

Uses `pterm.DefaultInteractiveTextInput` for these.

### UI component

All selectors use `pterm.DefaultInteractiveSelect` which is already a dependency (used in `cmd/snapshot.go`).

### Grouping

Existing section grouping is preserved:
1. **General**: project type, domain
2. **Language**: PHP version, Node version, package manager
3. **Database**: DB driver, DB version, DB name
4. **Services**: Redis version

### Scope boundaries

- `nova use php|node|db` commands remain positional args (not interactive)
- `config.Load()`, `config.LoadGlobal()`, and all other existing behavior unchanged
- No new external dependencies

## Files Changed

1. `internal/config/cmd/fetchversions/main.go` (new) - Docker Hub tag fetcher
2. `internal/config/gen_versions.go` (new) - `//go:generate` directive
3. `internal/config/versions_generated.go` (new, committed) - generated version slices
4. `cmd/init.go` (modified) - replace `prompt()` with interactive selectors
